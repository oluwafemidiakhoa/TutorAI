# TutorX-MCP Server

A comprehensive Model Context Protocol (MCP) server for educational AI tutoring, featuring a user-friendly Gradio web interface.

## Overview

TutorX-MCP is an AI tutoring platform that leverages the Model Context Protocol (MCP) for a robust backend and Gradio for the user interface. It provides a range of educational features, with a focus on AI-powered chat and quiz generation.

## Core Implemented Features

**🤖 Contextualized AI Tutoring:**
- **Session-based tutoring** with conversational memory, powered by Google Gemini.
- Interactive chat interface for real-time Q&A.

**🎨 AI-Powered Quiz Generation:**
- **Dynamic quiz creation** on any topic specified by the user.
- Customizable difficulty and number of questions.
- Quizzes are generated by Google Gemini.

## Getting Started

### Prerequisites

- Python 3.12 or higher
- Dependencies as listed in `pyproject.toml`.

### Installation

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/Meetpatel006/TutorX.git
    cd TutorX
    ```

2.  **Create a virtual environment and install dependencies:**
    ```bash
    # We recommend using uv for fast installation
    uv venv
    source .venv/bin/activate  # On Unix/macOS
    # .venv\Scripts\activate  # On Windows

    uv pip install -e ".[test]"
    ```

### Required API Keys

To use the AI-powered features, you need a Google AI API Key.

-   **Google AI API Key**: For Google Gemini model integration.

You can set this as an environment variable:

```bash
# Example for Unix/macOS
export GOOGLE_API_KEY="your-google-api-key"
```

*Note: The project includes placeholders for other services like Mistral OCR, but these are not yet implemented, so only the Google API key is required.*

### Running the Application

You can run the server and the user interface together with a single command:

```bash
python run.py
```

By default:
- The MCP server runs at `http://localhost:8000`
- The Gradio web interface runs at `http://127.0.0.1:7860`

Open your web browser and navigate to **`http://127.0.0.1:7860`** to use the application.

## MCP Tool Integration

The server exposes the following MCP tools that are fully implemented:

-   **AI Tutoring Tools** (`ai_tutor_tools.py`)
    -   `start_tutoring_session`: Starts a contextualized AI tutoring session.
    -   `ai_tutor_chat`: Handles interactive chat with the AI tutor.
-   **Quiz Tools** (`quiz_tools.py`)
    -   `generate_quiz_tool`: Creates AI-generated quizzes for specific concepts.

*Other tool files exist as placeholders for future development.*

## Project Structure

The project follows a modular structure to separate concerns between the server, tools, and UI.

```
tutorx-mcp/
├── app.py                   # Gradio web interface
├── run.py                   # Runner script for different modes
├── mcp_server/              # Core server implementation
│   ├── model/               # AI model integrations
│   │   └── gemini_flash.py  # Google Gemini integration
│   ├── tools/               # MCP tool implementations
│   │   ├── ai_tutor_tools.py
│   │   └── quiz_tools.py
│   └── ... (other placeholder tool files)
├── tests/                   # Test suite
│   └── test_tools_integration.py # Tests for implemented tools
├── docs/                    # Documentation (stubs)
├── pyproject.toml           # Project dependencies
└── README.md                # This file
```

## Testing

The project includes a test suite that can be run with:

```bash
# First, ensure you have installed the test dependencies
# uv pip install -e ".[test]"

python run_tests.py
```

The tests use mocking to verify tool logic without making live calls to the AI API.
